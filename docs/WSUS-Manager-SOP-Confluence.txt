h2. Purpose

WSUS Manager is a production-ready PowerShell automation suite for deploying, managing, and maintaining Windows Server Update Services (WSUS) backed by SQL Server Express 2022. Designed for both online and air-gapped (offline) network environments, it provides a centralized GUI for installing, configuring, operating, and monitoring WSUS servers — eliminating manual scripting for routine administration and enforcing consistent update policy across deployments.

h3. Problem Statement

Traditional WSUS workflows in air-gapped environments are slow, error-prone, and difficult to automate. A full metadata and content sync using the Windows Internal Database (WID) typically requires 3+ hours, largely due to:

* Slow WID restore operations
* Large, unoptimized metadata structures
* Manual re-approval of updates after every import
* Inefficient content transfer with no filtering
* Lack of automated declination logic

By migrating WSUS to SQL Server Express 2022, the entire process now completes in under 30 minutes, significantly reducing maintenance time and improving operational efficiency.

h3. Limitations of WID (Windows Internal Database)

* *Very slow imports* — WID restores often take hours
* *No rich metadata support* — prevents automation of declination logic
* *Difficult cleanup* — superseded and expired updates accumulate unchecked
* *Database stability issues* — WID performance degrades as size approaches ~10 GB
* *State loss* — update approvals and declines must be re-done after every import

----

h2. Benefits of SQL Server Express with WSUS

h3. Performance & Reliability

* Import/export completes in under 30 minutes
* Faster metadata processing
* Clean, consistent database structure for automation

h3. Update State Preservation

* Approved and declined updates are preserved across imports
* No need to manually re-approve after air-gap import

h3. Automation & Cleanup

* Automatic declination of superseded, expired, and stale updates
* Scheduled cleanup prevents the SQL Express 10 GB cap from being reached
* Reduces WsusContent folder footprint

h3. Content Efficiency

* Holds only required updates for configured products and classifications
* Prevents uncontrolled growth of the WsusContent directory
* Ensures fast, predictable air-gap transfers

h3. Workflow Comparison

||Workflow Aspect||Old Method (WID-Based WSUS)||New Method (SQL Export/Import)||
|Duration|>3 hours|<30 minutes total|
|Manual Effort|High (re-approvals, declines after every import)|Minimal (import + reset content)|
|Update States|Must re-approve and re-decline each cycle|Preserved automatically|
|Old Updates (>6 mo.)|Scripts could not auto-decline|Automatically declined during sync|
|Content Folder Size|Grows without limit|Trimmed via update cleanup routine|
|Database Risk|WID degrades near 10 GB|SQL Express capped; cleanup routine manages growth|

h2. Scope

*In scope:*

* Installation of WSUS with SQL Server Express 2022 (including migration from Windows Internal Database)
* Monthly online sync operations — update download, product classification, auto-decline, and auto-approval
* Air-gap transfer workflow — export to media, physical transport, import, and database restore
* Client targeting via Group Policy Objects deployed to Domain Controllers
* Scheduled maintenance (index rebuild, database cleanup, shrink)
* Health monitoring, diagnostics, and database size trending
* Database backup and restore operations

*Out of scope:*

* Windows Server OS installation, hardening, or domain join
* Active Directory domain setup or OU structure (covered by separate DC SOP)
* Patch testing or change management approval workflows
* Update approval for products outside the configured default list

h2. Definitions

||Term||Definition||
|WSUS|Windows Server Update Services — Microsoft server role for centralized Windows update distribution|
|SUSDB|The SQL Server database used by WSUS to store update metadata and approval state|
|Air-Gap|A network with no direct internet connectivity; updates must be physically transferred via removable media|
|DTR|Data Transfer Request — formal approval required to move data across classified or collateral network boundaries|
|GPO|Group Policy Object — Active Directory policy applied to domain-joined computers to configure WSUS client settings|
|SQL Express|Free edition of SQL Server; used by WSUS Manager for SUSDB; has a 10 GB database size limit|
|sqlcmd.exe|SQL Server command-line tool used as automatic fallback when the PowerShell SqlServer module is unavailable|
|WID|Windows Internal Database — built-in lightweight database engine; automatically migrated to SQL Express during installation|
|Online Server|A WSUS server with internet access used to sync update metadata and content from Microsoft Update servers|
|Health Score|Composite 0–100 score displayed on the dashboard: Green (≥80), Yellow (50–79), Red (<50); based on services, database, sync status, disk space, and last operation result|
|Auto-Decline|Automatic rejection of update categories (superseded, expired, ARM64, legacy builds, preview) during sync|
|Auto-Approve|Automatic approval of update categories (Critical, Security, Definition Updates, Updates, Update Rollups) after sync|
|Deep Cleanup|Full database maintenance: decline superseded, purge obsolete records, rebuild indexes, shrink database|
|Differential Export|Export of only updates newer than N days; used to minimize USB transfer size during recurring air-gap syncs|

h3. Granting SQL Server Sysadmin Privileges

{note}Required for database backup/restore operations. Perform this on both the online and air-gapped WSUS servers.{note}

*Step 1: Connect to SQL Server*

# Launch SQL Server Management Studio (SSMS)
# Server type: Database Engine
# Server name: {{localhost\SQLEXPRESS}}
# Authentication: SQL Server Authentication
# Login: sa (or default admin account)
# Check *Trust Server Certificate*
# Click Connect

*Step 2: Add Login with Sysadmin Role*

# In Object Explorer, expand Security > Logins
# Right-click Logins > New Login...
# Click Search... to locate the account
# Click Locations... > select Entire Directory
# Enter domain group (e.g., {{DOMAIN\System Administrators}}) > OK
# Go to Server Roles page
# Check sysadmin > OK

*Step 3: Refresh Permissions*

# Log out of the WSUS server
# Log back in to refresh group membership

h3. Smart Update Policy

*Auto-Declined (removed from catalog):*

||Rule||Details||
|Expired / Superseded|Updates replaced by newer versions|
|Older than 6 months|By Microsoft release date - already-approved updates are preserved|
|ARM64|Not applicable to x64 lab environments|
|Legacy Windows builds|21H2, 22H2, 23H2|
|Preview / Beta|Pre-release updates|
|Edge non-stable|Dev Channel, Beta Channel, Extended Stable - keeps Stable + WebView2 only|
|Office legacy|Microsoft 365 Apps, Office 2019, Office LTSC 2021 - keeps Office 2024/LTSC 2024|
|WSL|Windows Subsystem for Linux|

*Kept but not auto-approved (require manual review):*
* 25H2 updates
* x86 / 32-bit updates (lab is x64 only)
* Feature upgrades

*Default Products:* Windows 11, Windows Server 2019, Microsoft Edge, Microsoft Defender Antivirus, Microsoft Defender for Endpoint, Office 2016, Microsoft 365 Apps, SQL Server 2022, Security Essentials.

*Default Classifications:* Critical Updates, Security Updates, Definition Updates, Updates, Update Rollups.

h2. Domain Controller Setup

{warning}Run on the Domain Controller, NOT on the WSUS server!{warning}

{warning}AIR-GAP ONLY: These GPOs direct all Windows Update traffic to the internal WSUS server and block Microsoft Update. Do NOT deploy on internet-connected systems.{warning}

h3. What the Script Does

||Step||Action||
|1|Auto-detect the domain|
|2|Delete and reimport all 3 GPOs from backup (removes stale registry values)|
|3|Create required OUs if they do not exist|
|4|Link each GPO to appropriate OUs|
|5|Push policy update to all domain computers via schtasks RPC (no WinRM required)|

h3. GPO 1: WSUS Update Policy

Configures Windows Update client behavior. Linked to all workstation/server OUs.

||Setting||Value||Description||
|WUServer|http://WSUS01:8530|WSUS server URL (auto-replaced by script)|
|UseWUServer|Enabled|Master switch - use intranet WSUS instead of Microsoft|
|DoNotConnectToWindowsUpdateInternetLocations|Enabled|Block direct internet updates (critical for air-gap)|
|AUOptions|4 - Auto download and install|Auto-download and install on schedule|
|ScheduledInstallTime|22:00|Install time|
|ConfigureDeadlineForQualityUpdates|7 days|Force-install deadline|
|ConfigureDeadlineForFeatureUpdates|7 days|Force-install deadline|
|ConfigureDeadlineGracePeriod|0 days|No grace period after deadline|

h3. GPO 2: WSUS Inbound Allow

Opens WSUS server firewall for inbound client connections. Linked to Member Servers\WSUS Server OU.

||Property||Value||
|Direction|Inbound|
|Protocol|TCP|
|Local Ports|8530, 8531|
|Profiles|Domain, Private|

h3. GPO 3: WSUS Outbound Allow

Opens client firewall to reach the WSUS server. Linked to Domain Controllers, Member Servers, and Workstations OUs.

||Property||Value||
|Direction|Outbound|
|Protocol|TCP|
|Remote Ports|8530, 8531|
|Profiles|Domain, Private|

h2. HTTPS Configuration (Optional)

{code:language=powershell}
# Interactive (recommended)
.\Scripts\Set-WsusHttps.ps1

# Non-interactive
.\Scripts\Set-WsusHttps.ps1 -CertificateThumbprint "1234567890ABCDEF..."
{code}

The script binds the certificate to port 8531, runs wsusutil configuressl, and exports the certificate to C:\WSUS\WSUS-SSL-Certificate.cer.

After enabling HTTPS, update the GPO:

{code:language=powershell}
.\Set-WsusGroupPolicy.ps1 -WsusServerUrl "https://WSUS01:8531"
{code}

h2. Troubleshooting

||Issue||Solution||
|Endless downloads / stuck content|Content path must be C:\WSUS (NOT C:\WSUS\wsuscontent)|
|Dashboard shows "Not Installed"|WSUS service not detected - click Install WSUS|
|Buttons greyed out|Operation already running, or WSUS not installed - wait or restart app|
|Clients not checking in|Run gpupdate /force, verify GPOs are linked, check firewall ports 8530/8531|
|GroupPolicy module not found|Run: Install-WindowsFeature GPMC|
|Database restore fails|Verify sysadmin privileges on SQL Server (see Prerequisites)|
|Database near 10 GB|Run Deep Cleanup; SQL Express hard limit is 10 GB|
|Sync stuck at 0%|Check DNS configuration - GUI runs a DNS preflight check before syncing|
|SqlServer module missing|v4.0.4 auto-falls back to sqlcmd.exe; no manual install needed|
|"Content is still downloading" after import|Click Reset Content (under Diagnostics) to run wsusutil reset|
|Script not found error|Verify Scripts/ and Modules/ folders are with WsusManager.exe|
