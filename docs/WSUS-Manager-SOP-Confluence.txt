{toc}

----

h2. Purpose

GA-WsusManager is a Windows PowerShell 5.1 / WPF administration suite for WSUS servers. It installs WSUS with SQL Server Express, restores approved transfer folders on air-gapped servers, runs online maintenance, deploys packaged GPOs, and provides diagnostics/repair workflows.

The release package contains the executable plus the scripts/modules it needs. Do not deploy the EXE alone.

----

h2. Supported Deployment Model

||Area||Current behavior||
|Application package|Download GA-WsusManager-v4.1.0.zip, extract the whole folder, run GA-WsusManager.exe as Administrator|
|WSUS install|Install WSUS with SQL Server Express 2022; WID is detected/removed when present|
|Air-gap restore|Move the approved transfer folder by approved USB/removable media, restore SUSDB, Robocopy content when needed, then run Diagnostics|
|GPO deployment|Copy the whole DomainController folder to the Domain Controller and run Set-WsusGroupPolicy.ps1 there|
|Online operations|Run Online Sync and Schedule Task only on servers intentionally allowed to reach Microsoft Update|
|Diagnostics|Run Diagnostics for health checks, auto-fixes, SQL login repair, IIS/content path checks, permissions, and service state|

h2. New WSUS Server Setup

# Copy GA-WsusManager-v4.1.0.zip to the server.
# Extract the whole package under C:\WSUS\, for example C:\WSUS\WsusManager\.
# Place SQLEXPRADV_x64_ENU.exe in C:\WSUS\SQLDB\.
# Right-click GA-WsusManager.exe and choose *Run as Administrator*.
# If the dashboard says WSUS is missing, click *Install WSUS*.
# Keep the content path at C:\WSUS unless the site standard requires another path.
# After install, run *Diagnostics*.

Diagnostics should confirm:

* SQL Server service is running
* WSUS Service is running
* IIS/W3SVC is running
* SUSDB is reachable
* Operator has SQL sysadmin access
* C:\WSUS permissions include WSUS/IIS/client download principals
* IIS /Content points to C:\WSUS\WsusContent

h2. Air-Gap Restore Workflow

Use this when an approved transfer folder has already been moved into the disconnected network by approved USB/removable media.

The transfer folder must contain:

{code}
ApprovedTransfer\
+-- SUSDB_YYYYMMDD.bak
+-- WsusContent\
{code}

Keep the SUSDB .bak and WsusContent tree from the same transfer snapshot.

||Step||Location||Action||
|1|Air-gapped WSUS server|Install GA-WsusManager and build the server if WSUS is missing|
|2|Air-gapped WSUS server|Copy the complete approved transfer folder to the server or attach the approved media directly|
|3|GA-WsusManager|Click *Restore DB* and select the SUSDB .bak|
|4|GA-WsusManager|Click *Robocopy* only if WsusContent still needs to be copied into C:\WSUS\|
|5|GA-WsusManager|Click *Reset Content* only if WSUS still reports files as downloading|
|6|GA-WsusManager|Run *Diagnostics* and review any auto-fix output|

Recommended Robocopy commands when doing the same transfer manually:

{code:language=powershell}
robocopy "E:\WSUS-Transfer" "C:\WSUS" /E /MT:16 /R:2 /W:5
robocopy "E:\WSUS-Transfer\WsusContent" "C:\WSUS\WsusContent" /E /MT:16 /R:2 /W:5 /XO
{code}

{note}Depending on your program, moving files across network boundaries may require a Data Transfer Request or similar approval. Follow local policy before connecting removable media.{note}

h2. Online Operations

Run Online Sync only on a server intentionally allowed to reach Microsoft Update.

||Profile||Use case||Operations||
|Full|Monthly connected-server maintenance|Sync, cleanup, ultimate cleanup, backup|
|Quick|Weekly connected-server maintenance|Sync, cleanup, backup|
|Sync Only|Metadata/product refresh|Sync and approval policy only|

Product sync is additive: selected products are added without removing existing subscriptions. Current defaults include Windows 11, Windows Server 2019, .NET Framework, Edge, Defender, Office 2016, SQL Server, Exchange 2019, and Visual Studio 2022.

The smart decline policy declines expired, superseded, old unapproved, ARM64, legacy Windows builds, Preview/Beta, Edge non-stable, x86 Edge/Office, and WSL updates. It auto-approves recent x64 Critical, Security, Definition, Updates, Update Rollups, and Service Packs while keeping Upgrades/25H2 for manual review.

h2. Domain Controller GPO Setup

There is no GUI Create GPO menu item. The packaged script and GPO backups are the source of truth.

# Copy the whole DomainController folder to the Domain Controller.
# Open an elevated Windows PowerShell prompt inside that copied folder.
# Run:

{code:language=powershell}
.\Set-WsusGroupPolicy.ps1
{code}

The script:

||Step||Behavior||
|1|Auto-detects the domain|
|2|Reimports the four packaged GPOs from backup|
|3|Reuses existing Member Servers or Member_Servers OU if present|
|4|Creates missing Member Servers, WSUS Server, and Workstations OUs when needed|
|5|Links each GPO to the correct OU|
|6|Prompts to move the WSUS server computer object to Member Servers\WSUS Server|
|7|Pushes policy by schtasks RPC; WinRM is not required|

Packaged GPOs:

||GPO||Target||Purpose||
|WSUS Update Policy - Servers|Domain Controllers and Member Servers|Point servers at the internal WSUS server|
|WSUS Update Policy - Workstations|Workstations|Point workstations at the internal WSUS server|
|WSUS Inbound Allow|Member Servers\WSUS Server|Allow inbound WSUS ports on the WSUS server only|
|WSUS Outbound Allow|Workstations, Member Servers, Domain Controllers|Allow outbound WSUS client traffic|

{warning}These GPOs are for air-gapped networks. Do not deploy them to systems that should use Microsoft Update directly.{warning}

h2. Diagnostics and Repair

Run Diagnostics after install, restore, Robocopy, GPO changes, SQL login changes, or permission repairs.

Diagnostics checks:

* SQL Server, WSUS Service, IIS/W3SVC
* SUSDB connectivity
* SQL sysadmin state for the operator
* WSUS and SQL firewall rules
* IIS content path
* C:\WSUS ACLs
* WSUS content state when IIS tooling is available

The content ACL auto-fix grants list/read/execute where required for:

* BUILTIN\IIS_IUSRS
* NT AUTHORITY\Authenticated Users
* WSUS service/IIS identities as detected

Use *Fix SQL Login* under Diagnostics if preflight reports that the current operator is not SQL sysadmin. The installer also attempts to grant the installing operator sysadmin access during setup.

h2. Troubleshooting

||Symptom||Action||
|Dashboard shows WSUS missing|Click *Install WSUS* before restore, Diagnostics, Robocopy, or Online Sync|
|Preflight fails SQL sysadmin|Run *Fix SQL Login* under Diagnostics, then retry|
|Content permission failure|Run *Diagnostics* and allow auto-fix; confirm IIS_IUSRS and Authenticated Users have list/read/execute|
|Updates still show downloading|Confirm WsusContent is present, run *Reset Content*, wait 5-10 minutes, then run *Diagnostics*|
|Clients scan but cannot download|Confirm GPO URL, firewall rules, IIS /Content path, and C:\WSUS ACLs|
|GPO script says OU missing|Use the v4.1.0 DomainController folder; the script creates missing OUs|
|Tray minimize hides the window|Use the tray icon/context menu to restore the app; disable tray minimize in Settings if not wanted|
