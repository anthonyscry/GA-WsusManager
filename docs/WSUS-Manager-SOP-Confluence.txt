h3. Granting SQL Server Sysadmin Privileges

{note}Required for database backup/restore operations. Perform this on both the online and air-gapped WSUS servers.{note}

*Step 1: Connect to SQL Server*

# Launch SQL Server Management Studio (SSMS)
# Server type: Database Engine
# Server name: {{localhost\\SQLEXPRESS}}
# Authentication: SQL Server Authentication
# Login: sa (or default admin account)
# Check *Trust Server Certificate*
# Click Connect

*Step 2: Add Login with Sysadmin Role*

# In Object Explorer, expand Security > Logins
# Right-click Logins > New Login...
# Click Search... to locate the account
# Click Locations... > select Entire Directory
# Enter domain group (e.g., {{DOMAIN\\System Administrators}}) > OK
# Go to Server Roles page
# Check sysadmin > OK

*Step 3: Refresh Permissions*

# Log out of the WSUS server
# Log back in to refresh group membership

h3. First-Time Setup

{note}If downloaded from the internet, unblock the files first.{note}

{code:language=powershell}
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
Get-ChildItem -Path "C:\\WSUS" -Recurse -Include *.ps1,*.psm1 | Unblock-File

h2. GUI Application

h3. Overview

WSUS Manager v4.0.4 includes a full WPF GUI application (WsusManager.exe) providing:

* *Dashboard* with auto-refresh (30-second interval) and Health Score (0-100)
* *Server Mode* toggle (Online vs Air-Gap) with context-aware menus
* *Live Terminal Mode* for external PowerShell window output
* *Operation buttons* that disable during execution
* *Log panel* showing real-time operation output (right-click: Copy All / Save to File)
* *History view* showing last 50 operations
* *Desktop notifications* on operation completion (toast/balloon/log fallback)
* *Startup splash screen* with 4-stage progress bar
* *Keyboard shortcuts*: Ctrl+D (Diagnostics), Ctrl+S (Sync), Ctrl+H (History), Ctrl+R/F5 (Refresh)
* *System tray icon* with health-colored indicator, minimize-to-tray support

h3. Dashboard Metrics

||Metric||Description||
|Server Status|WSUS service state|
|Database Size|Current SUSDB size (alerts at 8 GB+) with trend indicator|
|DB Trend|Days-until-full estimate using linear regression over 30 days|
|Last Sync|Most recent synchronization timestamp (green <7d, yellow 7-30d, red >30d)|
|Health Score|Weighted composite score 0-100 (Green 80+, Yellow 50-79, Red <50)|
|Update Count|Total updates in the SUSDB database|
|Clients|Number of registered WSUS clients|
|Scheduled Task|Next scheduled sync task status|

h3. Smart Update Policy

*Auto-Declined (removed from catalog):*

||Rule||Details||
|Expired / Superseded|Updates replaced by newer versions|
|Older than 6 months|By Microsoft release date - already-approved updates are preserved|
|ARM64|Not applicable to x64 lab environments|
|Legacy Windows builds|21H2, 22H2, 23H2|
|Preview / Beta|Pre-release updates|
|Edge non-stable|Dev Channel, Beta Channel, Extended Stable - keeps Stable + WebView2 only|
|Office legacy|Microsoft 365 Apps, Office 2019, Office LTSC 2021 - keeps Office 2024/LTSC 2024|
|WSL|Windows Subsystem for Linux|

*Kept but not auto-approved (require manual review):*
* 25H2 updates
* x86 / 32-bit updates (lab is x64 only)
* Feature upgrades

*Default Products:* Windows 11, Windows Server 2019, Microsoft Edge, Microsoft Defender Antivirus, Microsoft Defender for Endpoint, Office 2016, Microsoft 365 Apps, SQL Server 2022, Security Essentials.

*Default Classifications:* Critical Updates, Security Updates, Definition Updates, Updates, Update Rollups.

h2. Air-Gapped Network Workflow

{note}Depending on your program, transferring files into SAP or collateral spaces may require a Data Transfer Request (DTR). Check with your security team before physically moving media across network boundaries.{note}

h3. Workflow Overview

||Step||Location||Action||
|1|Online WSUS Server|Run *Online Sync* with export path set|
|2|Online WSUS Server|Run *Transfer > Export* to copy to USB/Apricorn|
|3|Physical Transfer|Transport USB drive to air-gapped network|
|4|Air-Gapped WSUS Server|Run *Transfer > Import* from USB|
|5|Air-Gapped WSUS Server|Run *Restore Database*|
|6|Air-Gapped WSUS Server|Run *Reset Content* (Diagnostics menu)|
|7|Domain Controller|Run .\\Set-WsusGroupPolicy.ps1 (one-time setup)|

+-- SUSDB_20260320.bak           (latest database backup)

{code}

h2. Domain Controller Setup

{warning}Run on the Domain Controller, NOT on the WSUS server!{warning}

{warning}AIR-GAP ONLY: These GPOs direct all Windows Update traffic to the internal WSUS server and block Microsoft Update. Do NOT deploy on internet-connected systems.{warning}

h3. What the Script Does

||Step||Action||
|1|Auto-detect the domain|
|2|Delete and reimport all 3 GPOs from backup (removes stale registry values)|
|3|Create required OUs if they do not exist|
|4|Link each GPO to appropriate OUs|
|5|Push policy update to all domain computers via schtasks RPC (no WinRM required)|

h3. GPO 1: WSUS Update Policy

Configures Windows Update client behavior. Linked to all workstation/server OUs.

||Setting||Value||Description||
|WUServer|http://WSUS01:8530|WSUS server URL (auto-replaced by script)|
|UseWUServer|Enabled|Master switch - use intranet WSUS instead of Microsoft|
|DoNotConnectToWindowsUpdateInternetLocations|Enabled|Block direct internet updates (critical for air-gap)|
|AUOptions|4 - Auto download and install|Auto-download and install on schedule|
|ScheduledInstallTime|22:00|Install time|
|ConfigureDeadlineForQualityUpdates|7 days|Force-install deadline|
|ConfigureDeadlineForFeatureUpdates|7 days|Force-install deadline|
|ConfigureDeadlineGracePeriod|0 days|No grace period after deadline|

h3. GPO 2: WSUS Inbound Allow

Opens WSUS server firewall for inbound client connections. Linked to Member Servers\\WSUS Server OU.

||Property||Value||
|Direction|Inbound|
|Protocol|TCP|
|Local Ports|8530, 8531|
|Profiles|Domain, Private|

h3. GPO 3: WSUS Outbound Allow

Opens client firewall to reach the WSUS server. Linked to Domain Controllers, Member Servers, and Workstations OUs.

||Property||Value||
|Direction|Outbound|
|Protocol|TCP|
|Remote Ports|8530, 8531|
|Profiles|Domain, Private|

h2. HTTPS Configuration (Optional)

{code:language=powershell}
# Interactive (recommended)
.\\Scripts\\Set-WsusHttps.ps1

# Non-interactive
.\\Scripts\\Set-WsusHttps.ps1 -CertificateThumbprint "1234567890ABCDEF..."
{code}

The script binds the certificate to port 8531, runs wsusutil configuressl, and exports the certificate to C:\\WSUS\\WSUS-SSL-Certificate.cer.

After enabling HTTPS, update the GPO:

{code:language=powershell}
.\\Set-WsusGroupPolicy.ps1 -WsusServerUrl "https://WSUS01:8531"
{code}

h2. Troubleshooting

||Issue||Solution||
|Endless downloads / stuck content|Content path must be C:\\WSUS (NOT C:\\WSUS\\wsuscontent)|
|Dashboard shows "Not Installed"|WSUS service not detected - click Install WSUS|
|Buttons greyed out|Operation already running, or WSUS not installed - wait or restart app|
|Clients not checking in|Run gpupdate /force, verify GPOs are linked, check firewall ports 8530/8531|
|GroupPolicy module not found|Run: Install-WindowsFeature GPMC|
|Database restore fails|Verify sysadmin privileges on SQL Server (see Prerequisites)|
|Database near 10 GB|Run Deep Cleanup; SQL Express hard limit is 10 GB|
|Sync stuck at 0%|Check DNS configuration - GUI runs a DNS preflight check before syncing|
|SqlServer module missing|v4.0.4 auto-falls back to sqlcmd.exe; no manual install needed|
|"Content is still downloading" after import|Click Reset Content (under Diagnostics) to run wsusutil reset|
|Script not found error|Verify Scripts/ and Modules/ folders are with WsusManager.exe|
